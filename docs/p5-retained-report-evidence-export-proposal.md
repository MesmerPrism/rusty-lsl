# P5 Retained Report Evidence Export Proposal

This frozen P52 lane adds one substantive crate-private Morphospace advisory export proposal whose sole source is a consuming P51 cursor-output bundle. The bundle contains the completed cursor, which remains the sole owner of the actual P50/P51 retained reports, and the exact cursor pages retained during exhaustion. There is no independent report-history input and no second history enumeration. A successful proposal preserves every report, page, outer evidence vector, nested proposal, snapshot, report, sample, vector, and allocation exactly once. Its manifest copies only existing `Copy` evidence facts from the validated pages.

Before allocation or manifest mutation, construction requires the cursor's exact completed facts: completed evidence equals total evidence, remaining evidence is zero, and there is no current position. It then validates the page count and every page's report index, page index, start, end, total, length, evidence, and deterministic position against the reports retained by that cursor. Incomplete, duplicated, missing, reordered, out-of-range, or foreign evidence fails transactionally and returns the unchanged cursor-output bundle.

Construction has explicit nonzero report, page, and evidence bounds. It uses fallible checked collection arithmetic and conversions, preflights every count and index, and reserves the exact manifest length with `try_reserve_exact`. Report-limit, page-limit, evidence-limit, arithmetic, conversion, allocation, and composition failures return the complete original cursor-output bundle with no externally visible partial proposal.

Manifest order is exactly cursor order: pages in the order emitted by the exhausted cursor and evidence in each page's retained order. Entries identify their exact report, page, page-evidence, and global export positions and retain the page's existing start, end, and total facts. The proposal does not duplicate report/history authority or synthesize evidence outside those cursor pages.

The proposal is caller-requested, default-inert, advisory, and non-applying. It is not serialized as or exposed through a public protocol, does not infer missing evidence, loss, continuity, or causality, and claims no liblsl equivalence. It has no root/runtime export or application path and acquires no Manifold, session, stream, transport, control, activation, admission, routing, device, oracle, recovery, discovery, or policy authority.
