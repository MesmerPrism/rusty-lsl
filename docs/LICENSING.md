# Licensing

Project-owned Rusty LSL source and documentation use
`AGPL-3.0-or-later`. The repository-level `LICENSE` contains the full license
text. New source files should carry an SPDX identifier where practical.

Compatibility observations do not transfer another implementation's license
to independently authored code, but copied, adapted, generated, or recorded
artifacts still require an explicit provenance and redistribution review.
Official liblsl is treated only as an MIT-licensed black-box interoperability
oracle and reference endpoint. It is not a source template or a default
production dependency. rLSL source is prohibited as an implementation input.

No dependency, vendored source, native library, generated binding, fixture, or
oracle output may enter the public tree until its exact license and notice
obligations are recorded in `THIRD_PARTY_NOTICES.md` and the provenance record.
